# ライブラリ整備用ガイド

このリポジトリのライブラリを整備する際の基本方針。

## 方針
- 既存の構造・命名規則・ヘッダ配置を尊重する
- 競技プログラミング用途なので、過度な抽象化や依存追加は避ける
- 変更は最小限にし、必要な範囲で整合性を取る
- ドキュメントやサンプルがある場合は更新と同期を意識する

## テスト
以下のコマンドでテストを実行できる:

```
scripts/verify.sh
```

Python 依存は `pyproject.toml` で管理する。検証コマンドは `scripts/verify.sh` 経由で実行し、必要なら `scripts/verify.sh all --jobs 20` のように `oj-verify` の引数を渡す。
